# 1️⃣ Теория: что такое Binary Search Tree (20–25 минут)

## 🔹 1.1. Определение

**Binary Search Tree (BST)** — это бинарное дерево, в котором выполняется свойство упорядоченности:

- все элементы **левого поддерева < узла**
    
- все элементы **правого поддерева > узла**
    

```
        8
       / \
      3   10
     / \    \
    1   6    14
```

Для каждого узла выполняется правило BST.

---

## 🔹 1.2. Основные операции

|Операция|Средняя сложность|Худшая|
|---|---|---|
|Поиск|O(log n)|O(n)|
|Вставка|O(log n)|O(n)|
|Удаление|O(log n)|O(n)|

Худший случай возникает, когда дерево вырождается:

```
1
 \
  2
   \
    3
     \
      4
```

→ дерево превращается в список.

---

## 🔹 1.3. Почему BST полезно

BST позволяет эффективно:

- искать элементы
    
- находить **min/max**
    
- находить **следующий элемент (successor)**
    
- получать **k-й элемент по порядку**
    
- выполнять **range queries**
    

---

## 🔹 1.4. Inorder traversal — ключевое свойство

Если обойти BST **inorder**:

```
left → node → right
```

мы получим **отсортированный массив**.

Это очень важное свойство BST.

# 2️⃣ Как реализуется BST в Python (20–25 минут)

Python **не имеет встроенного BST**, поэтому его реализуют через класс узла.

---

## 🔹 2.1. Узел дерева

```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right
```

Каждый узел содержит:

- значение
    
- ссылку на левое поддерево
    
- ссылку на правое поддерево
    

---

## 🔹 2.2. Поиск элемента

```python
def search(root, target):
    if not root:
        return None

    if target == root.val:
        return root
    elif target < root.val:
        return search(root.left, target)
    else:
        return search(root.right, target)
```

Сложность: **O(h)**  
где `h` — высота дерева.

---

## 🔹 2.3. Вставка

```python
def insert(root, val):
    if not root:
        return TreeNode(val)

    if val < root.val:
        root.left = insert(root.left, val)
    else:
        root.right = insert(root.right, val)

    return root
```

---

## 🔹 2.4. Удаление (3 случая)

Удаление — самая сложная операция.

### 1️⃣ Узел — лист

Просто удалить.

### 2️⃣ Один ребёнок

Подставить ребёнка вместо узла.

### 3️⃣ Два ребёнка

Нужно найти **inorder successor** (минимум справа).

```
      5
     / \
    3   7
       /
      6
```

Удаляем `5`.

Заменяем на `6`.

---

## 🔹 2.5. Поиск минимума

```python
def find_min(root):
    while root.left:
        root = root.left
    return root
```

# 3️⃣ Что происходит "под капотом" (10–15 минут)

BST — это:

- набор объектов
    
- соединённых **указателями**
    

Каждый узел хранит:

```
value
left pointer
right pointer
```

В памяти структура выглядит примерно так:

```
Node(8)
 ↙     ↘
Node(3) Node(10)
```

Нет массива — только ссылки.

---

## Высота дерева

Сложность операций зависит от **высоты**:

```
Balanced tree → O(log n)

Unbalanced → O(n)
```

Поэтому существуют:

- AVL trees
    
- Red-Black trees
    

Но в задачах LeetCode чаще используют обычные BST.

    def iterative(self, root):
        if not root: return True
        stack = [(root, -float('inf'), float('inf'))]
        while len(stack):
            node, left, right = stack.pop()
            if node.val <= left or node.val >= right: return False

class Solution:
    def insertIntoBST(self, root: Optional[TreeNode], val: int) -> Optional[TreeNode]:
        if not root: return TreeNode(val)
            
        cur, next = None, root
        while next:
            cur = next
            next = cur.left if val < cur.val else cur.right
        
        if val < cur.val: 
            cur.left = TreeNode(val)
        else: 
            cur.right = TreeNode(val)
            
        return root

    def isValidBST(self, root: Optional[TreeNode], left = float('-inf'), right = float('inf')) -> bool:
