# 1️⃣ Теория: что такое куча (15–20 минут)

## 🔹 1.1. Определение

**Куча (Heap)** — это структура данных, представляющая собой **почти полное бинарное дерево**, в котором выполняется свойство кучи:

- **Min-heap:** каждый родитель ≤ детей
    
- **Max-heap:** каждый родитель ≥ детей
    

Главное свойство:

> Самый приоритетный элемент всегда находится в корне.

---

## 🔹 1.2. Зачем нужна куча?

Когда нужно:

- Быстро получать минимум/максимум
    
- Поддерживать топ-K элементов
    
- Работать с приоритетами
    
- Обрабатывать элементы "в порядке важности"
    

---

## 🔹 1.3. Основные операции

|Операция|Сложность|
|---|---|
|Вставка|O(log n)|
|Удаление корня|O(log n)|
|Получить минимум/максимум|O(1)|
|Построение кучи из массива|O(n)|

---

## 🔹 1.4. Почему операции O(log n)?

Высота почти полного бинарного дерева = O(log n)

При вставке:

- Элемент добавляется в конец
    
- Поднимается вверх (sift up)
    

При удалении:

- Корень заменяется последним элементом
    
- Опускается вниз (sift down)

# 2️⃣ Как куча реализована в Python (20–25 минут)

## 🔹 2.1. Модуль `heapq`

В Python используется модуль:

```python
import heapq
```

Это **min-heap**, реализованный поверх обычного списка.

---

## 🔹 2.2. Представление под капотом

Куча хранится в массиве.

Для элемента с индексом `i`:

- Левый ребёнок: `2*i + 1`
    
- Правый ребёнок: `2*i + 2`
    
- Родитель: `(i - 1) // 2`
    

Это компактное хранение бинарного дерева в массиве.

---

## 🔹 2.3. Основные операции в Python

```python
heap = []

heapq.heappush(heap, x)     # O(log n)
heapq.heappop(heap)         # O(log n)
heap[0]                     # O(1) минимум
heapq.heapify(arr)          # O(n)
```

---

## 🔹 2.4. Max-heap в Python

Стандартная библиотека поддерживает только min-heap.

Чтобы сделать max-heap:

```python
heapq.heappush(heap, -x)
```

или использовать кортежи:

```python
heapq.heappush(heap, (-priority, value))
```

---

## 🔹 2.5. Важные детали

### ✔️ Heap не сортирует массив полностью

Он гарантирует только свойство кучи.

### ✔️ Нельзя эффективно удалять произвольный элемент

(нет O(log n) remove по значению)

### ✔️ Для сложных структур используют кортежи

Python сравнивает кортежи лексикографически:

```python
(priority, value)
```

## 🔹 Задача 1 (разогрев, 15–20 мин)

**Kth Largest Element in an Array**

Обсудить:

- Полная сортировка → O(n log n)
    
- Min-heap размера k → O(n log k)
    

Очень важный инсайт.

```

Учитывая целочисленный массив nums и целое число k, верните k-й наибольший элемент в массиве.

Обратите внимание, что это k-й по величине элемент в отсортированном порядке, а не k-й отдельный элемент.

Можете ли вы решить эту задачу без сортировки?

Example 1:
Input: nums = [3,2,1,5,6,4], k = 2
Output: 5

Example 2:
Input: nums = [3,2,3,1,2,4,5,5,6], k = 4
Output: 4

```

```

class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        heap = nums[:k]
        heapq.heapify(heap)
        
        for num in nums[k:]:
            if num > heap[0]:
                heapq.heappop(heap)
                heapq.heappush(heap, num)
        
        return heap[0]

```

## 🔹 Задача 2 (20–25 мин)

**Top K Frequent Elements**

Комбинация:

- hashmap + heap
    

Навык:

- Хранение (частота, число)
    
- Использование кучи ограниченного размера
    

```

Учитывая целочисленный массив nums и целое число k, верните k наиболее часто встречающихся элементов. Вы можете вернуть ответ в любом порядке.

Example 1:
Input: nums = [1,1,1,2,2,3], k = 2
Output: [1,2]

Example 2:
Input: nums = [1], k = 1
Output: [1]

Example 3:
Input: nums = [1,2,1,2,1,2,3,1,3,2], k = 2
Output: [1,2]

```

```

from collections import Counter
class Solution:
    def topKFrequent(self, nums: List[int], k: int) -> List[int]:
        count = Counter(nums) 
        h = []
        
        for el, freq in count.items():
            heapq.heappush(h, (freq, el)) 
            if len(h) > k:
                heapq.heappop(h)

        return [el[1] for el in h]

        



from collections import Counter
class Solution:
    def topKFrequent(self, nums: List[int], k: int) -> List[int]: 
        # O(1) time 
        if k == len(nums):
            return nums
        
        # 1. Build hash map: character and how often it appears
        # O(N) time
        count = Counter(nums)   
        # 2-3. Build heap of top k frequent elements and
        # convert it into an output array
        # O(N log k) time
        return heapq.nlargest(k, count.keys(), key=count.get) 

```

## 🔹 Задача 3 (если остается время)

**Find Median from Data Stream**

Разобрать идею двух куч:

- max-heap для левой части
    
- min-heap для правой
    

```

Медиана - это среднее значение в упорядоченном целочисленном списке. Если размер списка четный, среднее значение отсутствует, а медиана - это среднее из двух средних значений.

Например, для arr = [2,3,4] медиана равна 3.
Например, для arr = [2,3] медиана равна (2 + 3) / 2 = 2.5.
Реализуем класс MedianFinder:

Функция MedianFinder() инициализирует объект MedianFinder.
void addNum(int num) добавляет целое число num из потока данных в структуру данных.
двойная функция findMedian() возвращает медиану всех элементов на данный момент. Принимаются ответы в пределах 10-5 от фактического ответа.

Example 1:
Input
["MedianFinder", "addNum", "addNum", "findMedian", "addNum", "findMedian"]
[[], [1], [2], [], [3], []]
Output
[null, null, null, 1.5, null, 2.0]

Explanation
MedianFinder medianFinder = new MedianFinder();
medianFinder.addNum(1);    // arr = [1]
medianFinder.addNum(2);    // arr = [1, 2]
medianFinder.findMedian(); // return 1.5 (i.e., (1 + 2) / 2)
medianFinder.addNum(3);    // arr[1, 2, 3]
medianFinder.findMedian(); // return 2.0

```

```

class MedianFinder:
	 def __init__(self):
        """
        initialize your data structure here.
        """
        
        self.left = []
        self.right = []
        

    def addNum(self, num: int) -> None:
        if not self.left or num <= -self.left[0]:
            heapq.heappush(self.left, -num) # we use negative numbers to make it a max
        else:
            heapq.heappush(self.right, num)
            
        # rebalance here
        if abs(len(self.left) - len(self.right)) > 1:
            if len(self.left) > len(self.right):
                elt = -heapq.heappop(self.left) # negate to get it back to original
                heapq.heappush(self.right, elt)
            else:
                elt = heapq.heappop(self.right)
                heapq.heappush(self.left, -elt) # negate due to max heap


    def findMedian(self) -> float:
        
        # remember to negate the left values!
        
        if len(self.left) == len(self.right):
            return (-self.left[0] + self.right[0]) / 2
        elif len(self.left) > len(self.right):
            return -self.left[0]
        else:
            return self.right[0]

```
